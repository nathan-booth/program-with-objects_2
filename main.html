Lessons in Programming

Lesson 1: The Web and HTML

The Web

The World Wide Web is a collection of connected HTML documents. HTTP stands for hypertext transfer protocol and it defines the rules for transferring files among computers, servers, and software (like browsers) on the web.

HTML

HTML stands for hypertext markup language; it is the code behind HTML documents that tells computers and browsers what information to display. An element of HTML is composed of an opening tag, content, and a closing tag, except in the case of void tags which do not require a closing tag. Elements are either inline or block; the latter forms a box around the content inside. Tags are often assigned attributes which change the functionality of an element, such as color or creating a link.

Last Words

In general, don't worry about memorizing bits of code; anything you haven't memorized by common use is easily found with a Google search or common resources, like w3schools or Stack Overflow.

Lesson 2: Structure and Tools

Document Object Model (DOM)

The DOM refers to the tree-like or nested structure of HTML documents. A web page can be viewed as a series of nested boxes in which content resides. Properly structured code uses indentation to reflect the organization of the boxes.

Tools

Chrome and other browsers have developer tools that can be used to inspect the code behind a web page.

Text editors like Sublime Text are used to write code because they make life for a programmer a bit easier by automatically completing bits of code, identifying syntax errors, and more.

Lesson 3: CSS and Philosophy

CSS

CSS stands for cascading style sheets and is the code that works in conjunction with HTML to affect the look or style of HTML documents. (Side note: CSS is usually used as a singular noun.) This is accomplished by using classes in HTML that CSS can use to affect everything of the same class plus the child elements of that class (see: inheritance).

On Repetition

Code is used to describe and execute patterns; in programs, it automates repetitious tasks. For static objects, like many documents, programmers still like to avoid repetition in order to maintain consistent code and avoid errors. In the context of HTML and CSS, using classes to make style changes means that coders need only write a code for a class and not rewrite code each time a particular effect is needed. This reduces the chance for minor errors and simplifies the debugging process.

Lesson 4: Automation

Computers and Programming

A computer is a machine that can execute a program. A program describes a sequence of steps for the computer to execute. Programs are written in a programming language, which is a constructed language for writing computer programs. Python is a programming language. Python uses an interpreter, which allows programmers to use this high-level language to write in a way that more closely resembles human written language.

An expression is something that has a value. Expressions use combinations of data types, operators, variables, and other expressions. Basic data types include integers (int), floating-point numbers (float), strings (str), Boolean values (Bool), and lists (list). Operators include arithmetic (addition, subtraction, multiplication, exponentiation, division, modulo) and logical (and, or, not, equal to, less than, greater than, etc.) operators. A variable stores a value on the computer. Assignment refers to associating some value to a variable. Beware the dangers of aliasing, when a value is referenced in multiple locations, because changing a value in one location changes it for every case associated with that location.

Strings and Lists

Strings are immutable. Lists are mutable and structured, which generally makes them more powerful than strings. Both data types are indexed similarly.

Functions

A function is like a sub-program that executes a specific task within a larger program. Judicious use reduces the amount of code a programmer must write and allows a program to be built as a series of sub-programs that can be built, tested, and re-used. This simplifies the task of writing complex programs. Functions take some input(s) or parameter(s) and return some output(s).

Control Flow

Control flow refers to specifying the order that tasks are executed. Some tasks need to be repeated until some endpoint is reached or condition is met; for and while are used, respectively, sometimes in conjunction with break. Some tasks only take place if a certain condition is met; these are conditional statements like if, then, else. If a condition is met, then the code in the block will be executed.

Debugging

Examine error messages. Work from example code and understand the desired inputs and outputs. Test examples. Check intermediate results with the prodigious use of print statements. Record and compare previous versions and results. To help minimize the debugging process, break down problems into simple pieces that can be tested individually and fit together.

Lesson 5: Object-Oriented Programming

OOP

Object-Oriented Programming (OOP) in Python refers to a class-based (hierarchical) system that organizes data structures. OOP is useful for increasing intuitive data organization, code reusability, and decomposing large programs into subprograms, which can be built directly. The decomposition process is similar to creating multiple procedural functions in Python to solve a complex computational problem.

Class inheritance functions identically in Python and CSS; it has the property of passing on its characteristics or behavior to child classes and instances. Classes share attributes that can be called repetitively. Subclasses have precedence over superclasses and instances have precedence over classes.

Classes and Objects

A class defines an archetype of an object. An object (or instance) of a class is a specific version of that class, like my dog Watson is an object of class Dog. A constructor is used to initialize a class; Python uses the __init__ function name within a class definition. __init__ defines the attributes of the class. Classes allow programmers to create new data types and can be re-used.

Methods and Objects

Instance variables exist within a particular instance of a class. Scott Adams, who belongs to the class Human, is assigned male to the instance variable gender. This is separate from the class variable gender, which allows for several possible designations, including male and female. Class variables are assigned at the class level of code, not the function level within a class.

Methods are functions associated with a particular class. Instance methods are functions within a class definition that control that class's behavior. Method overriding happens when subclass methods are executed over superclass methods of the same name, parameters, and return type.

Collections and Abstraction

A library is a large collection of related modules and code. Modules are collections of related functions and classes built for specific purposes. Modules are written in self-contained files that programs can call upon at need, similar to HTML documents calling CSS or JavaScript files to fulfill styling and dynamic function, respectively. Abstraction refers to how code details are hidden in order to manage the complexity of the code, be they Python or CSS classes or the HTML DOM. The details of methods are hidden to emphasize their functionality to users over their construction.
